API Integration with Payment Gateway Failing

The payment gateway integration is currently failing when users attempt to make purchases above $500. This is affecting approximately 15% of our transactions.

Steps to reproduce:

  1. Create a cart with items totaling more than $500
  2. Proceed to checkout
  3. Enter valid payment information
  4. Click "Submit Payment"

Expected behavior: Payment processes successfully

Actual behavior: Error message displays "Transaction failed: Code 4302"

Error logs show the following:

2025-03-05T14:32:11 ERROR [PaymentService] - Failed to process transaction
Request ID: tx-9876543
Error Code: 4302
Message: Transaction limit exceeded
Trace ID: 8a7b6c5d4e3f2g1h
